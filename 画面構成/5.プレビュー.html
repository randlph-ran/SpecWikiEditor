Markdown プレビュー

  • エディタのテキストが変更されるたびに、Markdig で HTML に変換し webView2Preview にリアルタイム表示する。
  • Markdig の拡張機能 UseSoftlineBreakAsHardlineBreak() を有効化しており、エディタ内の単一の改行(Enter1回)がそのまま <br> としてプレビュー・出力 HTML に反映される(Markdown 標準では単一改行は無視され、空行や行末の半角スペース2つが必要になるため、それを避けるための設定)。
  • 画像はプレビュー・出力 HTML のどちらでも、表示領域の幅に収まるよう CSS (max-width: 100%; height: auto;) で自動縮小される。画像ファイル自体はオリジナルサイズのまま保持され、加工されない。
  • HTML には背景色・文字色を明示指定しており、Windows のダークモード設定によって WebView2 が自動的に色反転してしまう(黒背景に黒文字になる)問題を防止している。加えて CoreWebView2Profile.PreferredColorScheme を Light に固定し、プレビューは常にライト表示となる。

画像のドラッグ&ドロップ

  • エディタに画像などのファイルをドラッグ&ドロップすると、GUID を付与したファイル名で assets フォルダにコピーする。
  • カーソル位置に、仮想ホスト名を使った Markdown 画像リンク(![画像](../assets/xxxxx.png))を自動挿入する。
  • プレビュー表示時は SetVirtualHostNameToFolderMapping により、この仮想ホスト URL を assets フォルダに解決している。
